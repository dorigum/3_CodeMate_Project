### 8. 배포 트러블슈팅

#### Docker Socket 권한 오류

1. 증상: `/var/run/docker.sock` 접근 시 `permission denied`.
2. 원인: Ubuntu 사용자의 Docker 그룹 권한 미적용.
3. 해결: 사용자를 `docker` 그룹에 추가하고 SSH 재접속.

#### Docker Hub Pull 거부

1. 증상: 예시 이미지 경로에 대한 `pull access denied`.
2. 원인: `.env.prod`의 `DOCKER_IMAGE`가 `docker-hub-username/codemate`로 남아 있음.
3. 해결: `dorigum/codemate`로 변경 후 Pull 재실행.

#### 컨테이너 이름 충돌

1. 증상: `codemate-mysql` 이름이 이미 사용 중이라는 오류.
2. 원인: `~/3_CodeMate`와 `~/codemate`에서 동일 Compose 컨테이너 이름 사용.
3. 해결: 운영 실행 위치를 CD 기준 디렉터리인 `~/codemate`로 통일.

#### CD SCP 연결 실패

1. 증상: `Copy deployment files to EC2` 단계에서 연결 시간 초과.
2. 원인: SSH `22` 인바운드를 개인 IP로만 제한해 GitHub Hosted Runner 접근 차단.
3. 해결: 실습 배포 중 Runner 접근 허용 후 CD 성공 확인.
4. 개선: 실제 운영에서는 SSM, 고정 Runner 또는 제한된 네트워크 사용 필요.

#### ALB 502 오류

1. 증상: 도메인 Health Check 요청에서 `502 Bad Gateway`.
2. 원인: Target Group이 애플리케이션 포트 `8080`이 아닌 `80`으로 연결.
3. 해결: HTTP `8080` Target Group 생성 및 EC2 대상 재등록.

#### Target 상태 Unused

1. 증상: 대상 등록 후 상태가 `Unused`.
2. 원인: Target Group이 ALB Listener에 연결되지 않음.
3. 해결: Listener Forward 규칙에 Target Group 연결.

#### Target Health Check 401

1. 증상: Target 상태 `Unhealthy`, 상세 코드 `401`.
2. 원인: 인증이 필요한 경로 또는 잘못된 기본 경로로 Health Check 요청.
3. 해결: 경로를 인증 없이 허용된 `/actuator/health`로 수정.
